# M4 Open-source Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the factory safe and understandable for public use with privacy gates, bilingual onboarding, validated examples, read-only CI, and separately verified GitHub publication.

**Architecture:** Treat tracked files as the only public surface, fail closed on private workspace paths or secret-like content, generate examples through the same production APIs users run, and let CI reproduce local gates without write permissions. Publication is a separate truth layer verified by remote SHA and checks.

**Tech Stack:** Python 3.11+, pytest, Markdown, Git, GitHub Actions using `actions/checkout@v6` and `actions/setup-python@v6`.

- [ ] **Step 3: Implement redacted findings**

```python
@dataclass(frozen=True, slots=True)
class PrivacyFinding:
    path: str
    code: str
    line: int | None
    fingerprint: str


@dataclass(frozen=True, slots=True)
class PrivacyReport:
    ok: bool
    findings: tuple[PrivacyFinding, ...]

    def to_json(self) -> str:
        return json.dumps(asdict(self), ensure_ascii=False, sort_keys=True)


```

Implement `scan_public_tree(root: Path, tracked_paths: Iterable[str]) -> PrivacyReport`. Reject `workshop/jobs/`, `workshop/reviews/`, `.env`, private keys, PEM blocks, GitHub token prefixes, common cloud access key formats, bearer authorization values, and files over the configured public size limit. Findings contain only path, line, code, and SHA256 fingerprint of the match; never the matched value. Skip binary contents but still enforce private paths and file size.

`scripts/verify_public.py` gets tracked paths from `git ls-files -z`, calls the scanner, prints redacted JSON, and exits `1` on findings.
