# 指挥官 Agent 工厂：客户快速使用指南

## 这是什么

它是一套 Agent 生产工具，可以把一个业务目标整理成结构完整、边界清楚、可以审查和测试的 Agent 工程包。

它有三种模式：

- `CREATE`：创建一个新 Agent。
- `REVIEW`：只读审查已有 Agent，不修改原文件。
- `OPTIMIZE`：先提供优化计划，经你明确批准后生成候选副本。

## 第一次安装

需要 Python 3.11 或更高版本。在终端进入本文件夹后执行：

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -e '.[dev]'
python -m factory.cli --version
python -m factory.cli verify-repo .
```

Windows PowerShell 激活虚拟环境的命令是：

```powershell
.venv\Scripts\Activate.ps1
```

看到 `commander-factory 0.1.0`，并且自检没有失败项，就可以开始使用。

## 最简单的方式：在 Codex 中使用

让 Codex 打开本文件夹，然后复制下面任意一段提示词。

### 创建新 Agent

```text
使用本项目的 commander-agent-factory，帮我创建一个 Agent。
请走 CREATE 流程，一次只问我一个最重要的问题。
不要猜测我的业务信息、数据来源、工具或权限。
等我明确确认完整需求后，再生成 Agent 工程包并告诉我输出位置和验证结果。
```

如果目标已经比较明确，可以在第一句后面补充，例如：

```text
这个 Agent 要帮助区域经理分析经销商的销售、库存和回款异常，并生成需要人工确认的行动建议。
```

### 审查已有 Agent

```text
使用本项目的 commander-agent-factory，按 REVIEW 流程只读审查这个 Agent：<填写 Agent 文件夹路径>。
不要修改原文件。请生成审查报告，并分别列出证据、主要缺口、严重程度和建议处理方式。
```

### 优化已有 Agent

```text
使用本项目的 commander-agent-factory，按 OPTIMIZE 流程优化这个 Agent：<填写 Agent 文件夹路径>。
先根据审查证据给我优化计划；未经我明确批准，不要创建候选副本，也不要修改原 Agent。
```

## 工厂会问哪些问题

CREATE 通常会逐步确认：

1. 要解决的业务问题。
2. 使用者和使用场景。
3. 期望结果与成功标准。
4. Agent 的核心任务。
5. 可使用的数据、知识和工具。
6. 允许与禁止的动作。
7. 必须人工审批的环节。
8. 用来验收 Agent 的测试案例。

信息不足时，工厂会继续提问，不会直接生产一个看似完整但未经确认的 Agent。

## 生成结果在哪里

多步骤任务默认放在本项目的 `workshop/` 中。CREATE 的典型结果包括：

- `COMMANDER_INTENT.md`：使命、目标、成功标准和权限边界。
- `AGENT_SPEC.yaml`：机器可读的 Agent 规格。
- `ARCHITECTURE.md`：能力、资源与工具结构。
- `WORKFLOW.md`：执行步骤和人工审批点。
- `evaluation/cases.yaml`：正常、失败、越界和未知情况的测试案例。
- `factory-manifest.json`：生成清单和追溯信息。

`workshop/jobs/` 和 `workshop/reviews/` 默认不会进入 Git。不要把密码、API Key、验证码或未脱敏客户数据写入准备公开或外发的文件。

## 如何理解完成状态

- 已生成：本地已经产生完整工程包。
- 已验证：本地校验和相关测试已经通过。
- 已安装：Agent 已放到目标运行环境。
- 已发布：目标远端或正式版本已经包含它。
- 已真实使用：真实用户已经运行，并取得可核验结果。

这些状态必须分别确认。生成成功不等于已经安装、发布或投入真实业务。
